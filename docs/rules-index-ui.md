# 📒 出荷ルール一覧画面の表示情報（原子リストと簡易モック）

## 9. 表示情報の原子リスト（再配置しやすい形）
「どのブロックに置くか」を柔軟に検討できるよう、情報粒度を 1 項目ずつに分解。不要と判断した項目はリストから除外済み。

| 情報項目 | 内容 / 例 | 現在の出現箇所（参考） | 表示条件・備考 |
| --- | --- | --- | --- |
| ページタイトル | 出荷ルール（配送エリア別） | ページヘッダー | 常時 |
| グローバルメッセージ | success/critical のテキスト | ページ上部 | 保存後のメッセージなど |
| 追加ボタン | 「配送エリアを追加」 | ページヘッダー | 常時 |
| 追加モーダル | 配送エリア選択 + 「設定する」 | モーダル | 未追加の配送エリアのみ選択可能 |
| テーブル: 配送エリア | 例: 北海道 | 一覧テーブル | 行クリックで詳細へ遷移 |
| テーブル: 対象配送ケース | 例: 3 件 | 一覧テーブル | 配送エリア内の配送ケース数 |
| テーブル: 基本設定（日） | 例: 3 日 / 未設定 | 一覧テーブル | `targetType === "all"` の日数 |
| テーブル: 商品別設定 | 例: 2 件 | 一覧テーブル | `targetType === "product"` の件数 |
| 空状態 | 「まだ配送エリアが追加されていません。」 | 本体エリア | 設定済み配送エリアが 0 件のとき |

### シンプルモック（原子リストのみで構成）
```
出荷ルール（配送エリア別）                         [配送エリアを追加]
──────────────────────────────────
[success/critical メッセージがある場合ここに表示]

配送エリアがない場合:
┌─────────────────────────────┐
│ まだ配送エリアが追加されていません。            │
│ [配送エリアを追加]                              │
└─────────────────────────────┘

配送エリアがある場合の例（行クリックで詳細へ）:
┌──────────────────────────────────────────────────────────┐
│ 配送エリア | 対象配送ケース | 基本設定（日） | 商品別設定 │
├──────────────────────────────────────────────────────────┤
│ 北海道   | 3 件            | 3 日           | 2 件       │
│ 沖縄     | 1 件            | 未設定         | 0 件       │
└──────────────────────────────────────────────────────────┘
```

## 10. 実装改善メモ（読みやすさ・現行コードからの変更点）
- 仕様: 最初から全配送ケースは表示せず、配送エリアを追加してから設定する。
- データ整形（loader 内）: 配送エリアごとに「対象配送ケース数」「基本設定（日）」「商品別設定件数」を集計して返す。
- 追加モーダル: 未追加の配送エリアのみ表示（追加済みは選べない）。
- 空状態: 「まだ配送エリアが追加されていません。」＋追加ボタン。
- 遷移: 行クリックで `/app/rules/:zoneKey` へ（`zoneKey` は配送エリア名から生成）。
