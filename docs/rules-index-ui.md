# 📒 出荷ルール一覧画面の表示情報（原子リストと簡易モック）

## 9. 表示情報の原子リスト（再配置しやすい形）
「どのブロックに置くか」を柔軟に検討できるよう、情報粒度を 1 項目ずつに分解。不要と判断した項目はリストから除外済み。

| 情報項目 | 内容 / 例 | 現在の出現箇所（参考） | 表示条件・備考 |
| --- | --- | --- | --- |
| ページタイトル | 出荷ルール（配送ケース別） | ページヘッダー | 常時 |
| グローバルメッセージ | success/critical のテキスト | ページ上部 | actionData あり時 |
| 配送ケース名 | 例: 北海道 | カードヘッダー | 常時 |
| 基本設定: 最終更新日時 | 例: 2024-03-01 12:34 | 基本設定ブロック | updatedAt を日付・時刻で表示 |
| 基本設定: 出荷リードタイム | 日数（例: 3 日） | 基本設定ブロック | 基本設定があるとき |
| 個別設定概要 | 件数のみ（例: 個別設定 2 件） | カードボディ | 個別設定があるとき。件数などの概要のみ表示 |
| 基本設定: 未設定メッセージ | 「基本設定がありません。」 | 基本設定ブロック | 基本設定なしのとき |
| 空状態: 配送ケースなし | 「配送ケースがありません。」 | 本体エリア | 配送ケース 0 件時のプレースホルダ |

### シンプルモック（原子リストのみで構成）
```
出荷ルール（配送ケース別）
──────────────────────────────────
[success/critical メッセージがある場合ここに表示]

配送ケースがない場合:
┌─────────────────────────────┐
│ 配送ケースがありません。                        │
└─────────────────────────────┘

配送ケースがある場合の例:
┌─────────────────────────────┐  （カード全体が詳細画面へのリンク）
│ 北海道                                           │
│ ・出荷リードタイム: 3 日                        │
│ ・個別設定: 2 件（個別設定がある場合）         │
│ ・最終更新: 2024-03-01 12:34                    │
└─────────────────────────────┘

┌─────────────────────────────┐
│ 沖縄                                              │
│ ・出荷リードタイム: 未設定の場合は               │
│   「基本設定がありません。」と表示              │
│ ・最終更新: 2024-02-20 09:15                    │
└─────────────────────────────┘
```

## 10. 実装改善メモ（読みやすさ・現行コードからの変更点）
- アクション/モーダルの整理: sync/toggle/delete/create 用 UI を撤去し、一覧は読み取り専用。カード全体を詳細画面への `<Link>` にする。
- データ整形（loader 内）: 配送ケースごとに「基本設定の日数・updatedAt」と「個別設定件数」だけを返すサマリ型に絞る。`targetType === "product"` 件数をサーバーで集計。
- 表示用フォーマット: `updatedAt` はサーバーで日付文字列に整形（例: YYYY-MM-DD HH:mm）。タイムゾーン明示を検討。
- 空状態テキスト: 「配送ケースがありません。」のみ。同期ボタン誘導は不要。
- スタイル: Polaris トークン/コンポーネントに寄せ、インライン style を減らす。`s-box` + `s-stack` で統一。
- 型の簡略化: `RuleItem` の不要フィールドを排除し、`baseDays`, `baseUpdatedAt`, `individualCount` など読みやすいフィールドに寄せる。
