# 出荷ルール一覧画面（配送エリア別）

## 表示情報（原子リスト）

| 情報項目 | 内容 / 例 | 出現箇所 | 表示条件・備考 |
| --- | --- | --- | --- |
| ページタイトル | 出荷ルール（配送エリア別） | ページヘッダー | 常時 |
| 成功メッセージ | 成功時のトースト | ページ上部 | `message` クエリが success のとき |
| エラーメッセージ | クリティカルバナー | ページ上部 | `message` クエリが critical のとき |
| 全体設定未完了バナー | 設定未完了案内 | ページ上部 | `defaultLeadDays` が未設定のとき |
| 追加ボタン | 「配送エリアを追加」 | ページヘッダー | 常時（設定未完了時は disabled） |
| 追加モーダル | 配送エリア選択 + 「設定する」 | モーダル | 未追加の配送エリアのみ選択可能 |
| 追加不可メッセージ | 追加できる配送エリアがありません。 | モーダル内 | 追加候補が 0 件のとき |
| テーブル: 配送エリア | 例: 北海道 | 一覧テーブル | 配送エリア名リンクで詳細へ遷移 |
| テーブル: 対象配送ケース | 例: 3 件 | 一覧テーブル | 配送エリア内の配送ケース数 |
| テーブル: 基本設定（日） | 例: 3 日 / 全体設定 (2 日) / 未設定 | 一覧テーブル | 基本設定があれば日数、なければ全体設定を表示 |
| テーブル: 商品別設定 | 例: 2 件 | 一覧テーブル | `targetType === "product"` の件数 |
| 行選択 | チェックボックス | 一覧テーブル | 複数選択で削除可能 |
| 一括削除ボタン | 「エリアを削除する」 | テーブル上部 | 選択件数 > 0 のとき有効 |
| 削除モーダル | 選択した配送エリアのルール削除 | モーダル | 基本設定・商品別設定を削除 |
| 空状態 | 「まだ配送エリアが追加されていません。」 | 本体エリア | 設定済み配送エリアが 0 件のとき |

## UIメモ
- 詳細遷移は「配送エリア名リンク」から行う（行クリックではない）。
- `defaultLeadDays` が未設定のとき、追加・削除・保存系操作は無効化される。
- `zoneKey` は配送エリア名から生成し、詳細 URL は `/app/rules/:zoneKey`。
